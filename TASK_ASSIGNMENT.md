# 电路模拟器开发任务分配

## 开发团队组成
- **OpenClaw**：主控制器，负责调度和监控
- **Hermes Agent 1**：前端交互开发
- **Hermes Agent 2**：算法核心开发

## 任务分配

### Hermes Agent 1 分工
```bash
# 获取任务
dnwxh -c ~/.hermes/config1.yaml

# 任务 1: UI设计
Issue #2: 任务1: 设计UI（电池、电阻、开关、电表图标）
预计工时: 2小时
关键技术: HTML Canvas, SVG, CSS Grid

# 任务 2: 拖拽连接
Issue #3: 任务2: 实现拖拽连接功能
逻耬时: 3小时
关键技术: HTML5 Drag & Drop API, Canvas事件处理

# 协助任务: 实时显示
Issue #5: 任务4: 添加电流/电压实时显示

```

### Hermes Agent 2 分工
```bash
# 获取任务
dnwxh -c ~/.hermes/config2.yaml

# 任务 3: 欧姆定律计算
Issue #4: 任务3: 编写欧姆定律计算逻辑
预计工时: 2.5小时
关键技术: 图论分析算法, 回谩计算, 错误处理
```

## 开发流程

### 阶段 1: 并行开发（第1天）
1. **OpenClaw 启动任务1和任务3**
   - Agent 1: 开始任务1 (UI设计)
   - Agent 2: 开始任务3 (计算逻辑)

2. **Agent 1 完成UI**后
   - 自动检测并通知OpenClaw
   - OpenClaw 启动任务2 (拖拽)

### 阶段 2: 集成开发（第2天）
3. **任务2完成**后
   - 集成任务3的计算逻辑
   - 开始任务4 (实时显示)

### 阶段 3: 合成测试（第3天）
4. **合成所有模块**
   - 全面测试电路连接、计算、显示
   - 修复bug

5. **Agent 1 完成任务5** (文档)

### 阶段 4: 部署（第4天）
6. **部署到GitHub Pages**
   - 自动化部署流程
   - 生成可分享链接

## 每日同步会议

### 08:00 早会
- 检查昨日进度
- 根据GitHub Issues状态更新任务排期
- 分配当日任务

### 12:00 午间同步
- 快速检查开发阻塞
- 协调需要合作的点

### 18:00 晚间总结
- 提交当日代码
- 更新GitHub Issues状态
- 创建第二天任务
